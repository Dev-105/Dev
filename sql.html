Built-in SQL Functions

Predefined | Powerful | Essential Operations

Comprehensive MySQL Function Reference

MySQL provides hundreds of built-in functions for data manipulation, calculation, and transformation. These functions are optimized for performance and can be used directly in SQL statements.

String Functions

CONCAT()

Concatenates strings together

SELECT CONCAT('Hello', ' ', 'World');
→ Hello World
SUBSTRING()

Extracts part of a string

SELECT SUBSTRING('MySQL Database', 1, 5);
→ MySQL
UPPER() / LOWER()

Changes case of string

SELECT UPPER('hello'), LOWER('WORLD');
→ HELLO, world
LENGTH() / CHAR_LENGTH()

Returns string length

SELECT LENGTH('MySQL');
→ 5 (bytes)
TRIM() / LTRIM() / RTRIM()

Removes spaces from string

SELECT TRIM(' Hello ');
→ Hello
REPLACE()

Replaces substring within string

SELECT REPLACE('Hello World', 'World', 'MySQL');
→ Hello MySQL

Numeric Functions

ROUND()

Rounds a number to specified decimals

SELECT ROUND(123.4567, 2);
→ 123.46
CEIL() / CEILING()

Rounds up to nearest integer

SELECT CEIL(123.456);
→ 124
FLOOR()

Rounds down to nearest integer

SELECT FLOOR(123.456);
→ 123
ABS()

Returns absolute value

SELECT ABS(-123.45);
→ 123.45
MOD()

Returns remainder of division

SELECT MOD(10, 3);
→ 1
POW() / POWER()

Raises a number to a power

SELECT POW(2, 3);
→ 8
SQRT()

Returns square root

SELECT SQRT(16);
→ 4
RAND()

Returns random number between 0 and 1

SELECT RAND();
→ Random value like 0.734

Date & Time Functions

NOW() / CURDATE() / CURTIME()

Current date and time

SELECT NOW(), CURDATE(), CURTIME();
→ Current datetime, date, time
DATE_ADD() / DATE_SUB()

Adds/subtracts time interval

SELECT DATE_ADD(NOW(), INTERVAL 7 DAY);
→ Date 7 days from now
DATEDIFF()

Difference between two dates in days

SELECT DATEDIFF('2024-12-31', '2024-01-01');
→ 365 days
TIMESTAMPDIFF()

Difference between two timestamps in specified unit

SELECT TIMESTAMPDIFF(YEAR, '2000-01-01', NOW());
→ Years between dates
YEAR() / MONTH() / DAY()

Extracts date parts

SELECT YEAR(NOW()), MONTH(NOW()), DAY(NOW());
→ Current year, month, day
DATE_FORMAT()

Formats date as specified

SELECT DATE_FORMAT(NOW(), '%W, %M %d, %Y');
→ Thursday, January 01, 2024
STR_TO_DATE()

Converts string to date

SELECT STR_TO_DATE('01,5,2023', '%d,%m,%Y');
→ 2023-05-01
DAYNAME() / MONTHNAME()

Returns name of day/month

SELECT DAYNAME(NOW()), MONTHNAME(NOW());
→ Monday, January

Aggregate Functions

COUNT()

Counts number of rows

SELECT COUNT(*) FROM users;
→ Total number of users
SUM()

Calculates sum of values

SELECT SUM(amount) FROM orders;
→ Total amount of all orders
AVG()

Calculates average of values

SELECT AVG(price) FROM products;
→ Average product price
MIN() / MAX()

Finds minimum/maximum value

SELECT MIN(price), MAX(price) FROM products;
→ Lowest and highest price
GROUP_CONCAT()

Concatenates values from group

SELECT category, GROUP_CONCAT(product_name) FROM products GROUP BY category;
→ All products in category as comma-separated list
STDDEV() / VARIANCE()

Statistical functions

SELECT STDDEV(price) FROM products;
→ Standard deviation of prices
Real-World Function Examples
-- Calculate customer age in years
SELECT 
    name,
    birth_date,
    TIMESTAMPDIFF(YEAR, birth_date, CURDATE()) AS age_years
FROM customers;

-- Format order dates nicely
SELECT 
    order_id,
    DATE_FORMAT(order_date, '%M %d, %Y at %h:%i %p') AS formatted_date,
    ROUND(total_amount, 2) AS amount
FROM orders;

-- Calculate days until subscription expires
SELECT 
    user_id,
    DATEDIFF(expiry_date, CURDATE()) AS days_remaining,
    CASE
        WHEN DATEDIFF(expiry_date, CURDATE()) <= 7 THEN 'Renew soon'
        WHEN DATEDIFF(expiry_date, CURDATE()) <= 0 THEN 'Expired'
        ELSE 'Active'
    END AS status
FROM subscriptions;

-- Generate username from email (extract part before @)
SELECT 
    email,
    SUBSTRING_INDEX(email, '@', 1) AS username,
    UPPER(SUBSTRING(email, 1, 1)) AS first_initial
FROM users;

-- Calculate weighted average with ROUND
SELECT 
    product_id,
    ROUND(SUM(quantity * price) / SUM(quantity), 2) AS avg_price
FROM order_items
GROUP BY product_id;
Best Practices for Using Functions
  • Performance: Functions in WHERE clauses can prevent index usage. Use computed columns instead for frequently filtered data.
  • Readability: Use aliases (AS) when functions return calculated values.
  • NULL handling: Most functions return NULL if any argument is NULL. Use IFNULL() or COALESCE() for defaults.
  • Data types: Be aware of implicit type conversions when mixing data types in functions.

Stored Procedures

Modular Code | Reusability | Security

What are Stored Procedures?

A stored procedure is a prepared SQL code that you can save and reuse. It's like a function in other programming languages.

Why Use Procedures?

  • Reduce network traffic (execute multiple statements at once)
  • Increase performance (compiled and stored in database)
  • Enhance security (control data access)
  • Promote code reusability

Parameter Types

IN Input parameter (default)
OUT Output parameter
INOUT Input and output parameter

Examples

-- Create a procedure with parameters
DELIMITER //

CREATE PROCEDURE GetCustomerOrders(
    IN customerId INT,
    OUT totalOrders INT
)
BEGIN
    -- Count orders for a customer
    SELECT COUNT(*) INTO totalOrders
    FROM orders
    WHERE customer_id = customerId;
    
    -- Return order details
    SELECT * FROM orders
    WHERE customer_id = customerId
    ORDER BY order_date DESC;
END //

DELIMITER ;

-- Call the procedure
CALL GetCustomerOrders(123, @total);
SELECT @total AS total_orders;
Real-World Usage

Use procedures for complex business logic like processing orders, generating reports, or data validation that involves multiple SQL statements.

User-Defined Functions

Return Value | Scalar | Deterministic

What are User-Defined Functions?

A user-defined function is a database object that returns a single value. Unlike procedures, functions can be used directly in SQL statements.

Function vs Procedure

Aspect Function Procedure
Return Value Must return a value Can return 0 or more values
SQL Statement Usage Can be used in SELECT Called with CALL
Transaction Cannot contain transactions Can contain transactions

Examples

-- Create a function to calculate age
DELIMITER //

CREATE FUNCTION CalculateAge(
    birthDate DATE
) 
RETURNS INT
DETERMINISTIC
BEGIN
    DECLARE age INT;
    
    SET age = YEAR(CURDATE()) - YEAR(birthDate);
    
    -- Adjust if birthday hasn't occurred this year
    IF MONTH(CURDATE()) < MONTH(birthDate) 
        OR (MONTH(CURDATE()) = MONTH(birthDate)
        AND DAY(CURDATE()) < DAY(birthDate))
    THEN
        SET age = age - 1;
    END IF;
    
    RETURN age;
END //

DELIMITER ;

-- Use the function in a SELECT statement
SELECT 
    name,
    birth_date,
    CalculateAge(birth_date) AS age
FROM users
WHERE CalculateAge(birth_date) > 18;

Variables

Data Storage | Temporary Values | Scoping

Variable Types in MySQL

Variables store temporary data during the execution of SQL code blocks.

Variable Scope

Local Variables

Declared inside procedures/functions with DECLARE. Only visible within the BEGIN-END block.

Session Variables

Start with @ (e.g., @total). Available throughout the session.

Global Variables

Start with @@ (e.g., @@global.max_connections). System-wide variables.

Examples

-- Using variables in a stored procedure
DELIMITER //

CREATE PROCEDURE CalculateOrderSummary(
    IN orderId INT
)
BEGIN
    -- Declare local variables
    DECLARE subtotal DECIMAL(10,2) DEFAULT 0.00;
    DECLARE taxRate DECIMAL(5,2) DEFAULT 0.08;
    DECLARE taxAmount, totalAmount DECIMAL(10,2);
    
    -- Calculate subtotal
    SELECT SUM(quantity * unit_price) 
    INTO subtotal
    FROM order_items
    WHERE order_id = orderId;
    
    -- Calculate tax and total
    SET taxAmount = subtotal * taxRate;
    SET totalAmount = subtotal + taxAmount;
    
    -- Return results
    SELECT 
        subtotal AS order_subtotal,
        taxAmount AS tax,
        totalAmount AS order_total;
END //

DELIMITER ;

Conditions

Control Flow | Decision Making | Business Logic

Conditional Logic in SQL

MySQL provides IF statements and CASE expressions to implement conditional logic.

IF vs CASE

IF Statement

Used in stored programs (procedures, functions, triggers). Similar to if-else in programming languages.

CASE Expression

Can be used in both stored programs and regular SQL queries. Two forms: simple and searched.

Logical Operators

AND
OR
NOT
XOR

Examples

-- IF statement in a procedure
DELIMITER //

CREATE PROCEDURE CheckInventory(
    productId INT,
    requestedQty INT
)
BEGIN
    DECLARE currentStock INT;
    
    SELECT stock_quantity INTO currentStock
    FROM products
    WHERE id = productId;
    
    IF currentStock >= requestedQty THEN
        SELECT 'Available' AS status;
    ELSEIF currentStock > 0 THEN
        SELECT 'Partial stock available' AS status;
    ELSE
        SELECT 'Out of stock' AS status;
    END IF;
END //

DELIMITER ;

-- CASE expression in a SELECT statement
SELECT 
    name,
    price,
    CASE
        WHEN price < 10 THEN 'Budget'
        WHEN price BETWEEN 10 AND 50 THEN 'Mid-range'
        ELSE 'Premium'
    END AS price_category
FROM products;

Loops

Repetition | Iteration | Batch Processing

Loop Structures in MySQL

Loops allow you to execute a block of SQL statements repeatedly until a condition is met.

Types of Loops

LOOP

Infinite loop that requires LEAVE statement to exit.

WHILE

Loop while a condition is true. Condition checked before each iteration.

REPEAT

Loop until a condition is true (do-while style). Condition checked after each iteration.

Control Statements

LEAVE

Exit the loop (like break)

ITERATE

Skip to next iteration (like continue)

Examples

-- Using a WHILE loop to generate test data
DELIMITER //

CREATE PROCEDURE GenerateTestOrders(
    customerId INT,
    numOrders INT
)
BEGIN
    DECLARE counter INT DEFAULT 1;
    DECLARE orderDate DATE;
    
    WHILE counter <= numOrders DO
        -- Set order date (past 30 days)
        SET orderDate = DATE_SUB(CURDATE(), 
            INTERVAL FLOOR(RAND() * 30) DAY);
        
        -- Insert test order
        INSERT INTO orders (customer_id, order_date, total_amount)
        VALUES (customerId, orderDate, 
            ROUND(50 + (RAND() * 200), 2));
        
        SET counter = counter + 1;
    END WHILE;
    
    SELECT CONCAT(numOrders, ' test orders created') AS result;
END //

DELIMITER ;

-- Using LOOP with LEAVE
-- (example skeleton)
DECLARE x INT DEFAULT 0;
myLoop: LOOP
    SET x = x + 1;
    IF x > 10 THEN
        LEAVE myLoop;
    END IF;
    -- Do something...
END LOOP myLoop;

Cursors

Row-by-row Processing | Result Set Navigation

What are Cursors?

A cursor is a database object used to retrieve and manipulate rows from a result set one row at a time.

When to Use Cursors

  • Row-by-row operations that can't be done with set-based SQL
  • Complex calculations on each row
  • When you need to process rows in a specific order
  • Batch processing with progress tracking

Cursor Lifecycle

1
DECLARE cursor
2
OPEN cursor
3
FETCH rows
4
CLOSE cursor

Examples

-- Using cursor to process orders row by row
DELIMITER //

CREATE PROCEDURE ProcessPendingOrders()
BEGIN
    DECLARE done INT DEFAULT 0;
    DECLARE orderId INT;
    DECLARE orderTotal DECIMAL(10,2);
    DECLARE customerEmail VARCHAR(255);
    
    -- Declare cursor for pending orders
    DECLARE orderCursor CURSOR FOR
        SELECT o.id, o.total_amount, c.email
        FROM orders o
        JOIN customers c ON o.customer_id = c.id
        WHERE o.status = 'pending'
        ORDER BY o.order_date;
    
    -- Declare continue handler for end of result set
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = 1;
    
    OPEN orderCursor;
    
    processLoop: LOOP
        FETCH orderCursor INTO orderId, orderTotal, customerEmail;
        
        IF done = 1 THEN
            LEAVE processLoop;
        END IF;
        
        -- Process each order (example: send email notification)
        -- In real scenario, you might update status, log, etc.
        INSERT INTO order_logs (order_id, action, timestamp)
        VALUES (orderId, 'Processed', NOW());
        
        UPDATE orders
        SET status = 'processed',
            processed_date = NOW()
        WHERE id = orderId;
        
    END LOOP processLoop;
    
    CLOSE orderCursor;
    
    SELECT 'Pending orders processed successfully' AS message;
END //

DELIMITER ;

Triggers

Automatic Execution | Data Integrity | Audit Logging

What are Triggers?

Triggers are stored programs that automatically execute in response to specific events on a table (INSERT, UPDATE, DELETE).

Trigger Timing

BEFORE Triggers

Execute before the operation. Used for validation, data transformation, or preventing invalid operations.

AFTER Triggers

Execute after the operation. Used for auditing, logging, or maintaining derived data.

NEW and OLD References

NEW.column Refers to new value (INSERT/UPDATE)
OLD.column Refers to old value (UPDATE/DELETE)

Examples

-- Create audit log trigger
DELIMITER //

CREATE TRIGGER audit_customer_changes
AFTER UPDATE ON customers
FOR EACH ROW
BEGIN
    INSERT INTO customer_audit (
        customer_id,
        changed_field,
        old_value,
        new_value,
        changed_by,
        change_date
    )
    VALUES (
        NEW.id,
        'email',
        OLD.email,
        NEW.email,
        USER(),
        NOW()
    );
END //

DELIMITER ;

-- BEFORE INSERT trigger for validation
DELIMITER //

CREATE TRIGGER validate_order_amount
BEFORE INSERT ON orders
FOR EACH ROW
BEGIN
    IF NEW.total_amount <= 0 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Order amount must be greater than 0';
    END IF;
    
    -- Auto-set order date if not provided
    IF NEW.order_date IS NULL THEN
        SET NEW.order_date = CURDATE();
    END IF;
END //

DELIMITER ;

SIGNAL SQLSTATE

Error Handling | Custom Errors | Validation

What is SIGNAL SQLSTATE?

The SIGNAL statement is used to return an error or warning condition from a stored program.

SQLSTATE Codes

'45000' - Unhandled User-defined Exception

Most commonly used for custom application errors.

'01000' - Warning

For non-critical notifications.

'HY000' - General Error

Generic error when no specific code applies.

SIGNAL vs RESIGNAL

SIGNAL Raise a new error condition
RESIGNAL Pass along an existing error after modification

Examples

-- Using SIGNAL for validation in a procedure
DELIMITER //

CREATE PROCEDURE UpdateProductPrice(
    productId INT,
    newPrice DECIMAL(10,2)
)
BEGIN
    DECLARE currentPrice DECIMAL(10,2);
    
    SELECT price INTO currentPrice
    FROM products
    WHERE id = productId;
    
    -- Validate price increase is not too large
    IF newPrice > currentPrice * 1.5 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Price increase cannot exceed 50%',
            MYSQL_ERRNO = 1644;
    END IF;
    
    -- Validate price is positive
    IF newPrice <= 0 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Price must be greater than 0';
    END IF;
    
    -- Update the price
    UPDATE products
    SET price = newPrice,
        last_updated = NOW()
    WHERE id = productId;
    
    SELECT 'Price updated successfully' AS message;
END //

DELIMITER ;

-- Trigger with SIGNAL for data integrity
CREATE TRIGGER prevent_archive_delete
BEFORE DELETE ON archived_orders
FOR EACH ROW
BEGIN
    SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'Cannot delete from archived_orders table';
END;

Built-in vs User-Defined Functions

Built-in Functions:

Predefined, optimized, no CREATE statement needed. Use for common operations.

User-Defined Functions:

Custom logic, CREATE FUNCTION required. Use when built-in functions don't meet your needs.

Function Categories Quick Reference

String Operations: CONCAT(), SUBSTRING(), REPLACE()
Date/Time: NOW(), DATE_ADD(), DATEDIFF()
Numeric: ROUND(), CEIL(), FLOOR(), RAND()
Aggregation: SUM(), AVG(), COUNT(), MAX()

Quick Decision Guide

  1. Need common operation (math, dates, strings)? → Use BUILT-IN FUNCTIONS
  2. Need automatic data validation/logging? → Use TRIGGER
  3. Need custom calculation reusable in queries? → Use USER-DEFINED FUNCTION
  4. Need complex business logic with transactions? → Use PROCEDURE
  5. Need row-by-row processing? → Use CURSOR inside a procedure
  6. Need custom error messages? → Use SIGNAL SQLSTATE
